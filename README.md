# ncsa-blacklist

Sync + lookup service for NCSA Thailand's public threat blocklists (IP / domain / SHA256 hash, MISP-sourced, updated daily, CC0). Provides a REST API, web UI, CLI, firewall/DNS/SIEM export formats, and a watch-list with webhook alerts.

## Run with Docker (recommended)

```sh
cp .env.example .env   # fill in ADMIN_TOKEN at minimum if exposing publicly
docker compose up -d
```

This starts two containers sharing one data volume:
- `ncsa-blacklist` — web UI + API on port 3939
- `ncsa-blacklist-sync` — daily cron sync of the feeds

First run has no data until the sync container completes its first fetch (or run `docker compose exec ncsa-blacklist node src/fetch.js` once manually).

## Run without Docker

```sh
npm install
npm run fetch       # initial data pull
npm start            # web UI + API on :3939
npm run schedule     # separate process: daily sync
```

## Configuration (env vars)

| Var | Default | Purpose |
|---|---|---|
| `PORT` | 3939 | HTTP port |
| `DATA_DIR` | `./data` | Where blocklist snapshots/history/watchlist are stored |
| `ADMIN_TOKEN` | unset | Single bearer token required for `/reload` and `/watch` writes (logged as admin `"default"`). **Set this (or `ADMIN_TOKENS`) before exposing the service beyond localhost.** |
| `ADMIN_TOKENS` | unset | Multiple named tokens, e.g. `alice:tok1,bob:tok2` — each request is logged with the matching name. Merges with `ADMIN_TOKEN` if both are set. |
| `WEBHOOK_URL` | unset | POSTed a sync summary (added/removed counts, watch-list hits, sync errors) after each sync |
| `WEBHOOK_SECRET` | unset | If set, signs the webhook body with HMAC-SHA256 in an `X-Signature: sha256=<hex>` header — verify it on your receiving endpoint if it's a custom one (not needed for Slack/Discord) |
| `CORS_ORIGIN` | unset | Allow cross-origin API calls from this origin |
| `RATE_LIMIT` | 60 | Max requests per minute per client IP |

## HTTPS

The service only speaks plain HTTP on `PORT`. Put a reverse proxy in front of it for TLS — example configs in `Caddyfile.example` and `nginx.conf.example` (just `reverse_proxy`/`proxy_pass` to `localhost:3939`).

## Backup

All state — blocklist snapshots, sync history, watch list — lives under `DATA_DIR` (the `ncsa-data` Docker volume, or `./data` on bare metal). Everything in it is reproducible except the watch list, since the blocklists themselves are just re-fetched from NCSA next sync. Back up `DATA_DIR/watchlist.json` if you have one configured; the rest can be regenerated by re-running `npm run fetch`.

## API

- `GET /stats` — current totals per feed type
- `GET /check/:type/:value` — `type` is `ip`, `domain`, or `hash`
- `POST /scan` (text body) — extracts IPv4s from pasted log text, flags any blacklisted
- `GET /history?type=ip` — last 30 sync runs
- `GET /watch` / `POST /watch` / `DELETE /watch` — `{ type, value }` body (writes require `ADMIN_TOKEN`)
- `POST /reload` — reload in-memory store from disk (requires `ADMIN_TOKEN`)
- `GET /healthz` — liveness check

## Export formats

```sh
npm run export:iptables   # ipset + iptables DROP rules
npm run export:dnsmasq    # dnsmasq sinkhole (address=/domain/0.0.0.0)
npm run export:wazuh      # CDB key:value list for Wazuh hash lookups
```
